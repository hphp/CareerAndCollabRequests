# 医院2026年公开招投标情报收集需求文档

> _版本：v1.0 | 日期：2026-04-10 | 编制：Eva_

---

## 1. 文档目的

本需求文档旨在建立一套面向全国各级医疗机构（公立医院、民营医院、基层医疗机构等）2026年度公开招投标信息的自动化情报收集与分析体系。覆盖具身智能、智慧医院、医疗信息化、医疗设备等领域的招标需求，为赢彻智能科技、采采木瓜智能科技及相关合作伙伴提供战略情报支持。

## 2. 情报收集范围

### 2.1 标的类型（Bid Targets）

| 序号 | 标的类别 | 说明 |
|------|---------|------|
| 1 | 具身智能医疗机器人 | 手术机器人、康复机器人、护理机器人、导诊机器人、消毒机器人等 |
| 2 | 智慧医院整体解决方案 | 智慧病房、智慧门诊、智慧后勤、智慧管理等 |
| 3 | 医疗信息化系统 | HIS、LIS、PACS、EMR、远程医疗、医疗大数据平台等 |
| 4 | 医疗设备采购 | 影像设备（CT/MRI/超声）、检验设备、监护设备、手术设备等 |
| 5 | 医院基础设施智能化 | 楼宇自动化、能耗管理、安全监控、智能停车等 |
| 6 | 医疗人工智能应用 | AI辅助诊断、智能影像分析、药物研发、智能随访等 |
| 7 | 医用耗材与试剂 | 集中带量采购、试剂配送等 |
| 8 | 医院新建/改扩建工程 | 智慧医院专项工程、信息化建设工程 |

### 2.2 地域范围

- **优先级A（重点）**：全国三甲医院、省级三甲医院
- **优先级B（次重点）**：地市级医院、县级医院
- **优先级C（补充）**：基层医疗机构（社区卫生服务中心、乡镇卫生院）
- **重点区域**：华东（上海、江苏、浙江）、华北（北京、天津）、华南（广东）、华中（湖北、湖南）、西部（四川、陕西、宁夏、甘肃）

### 2.3 时间范围

- 2026年1月1日 — 2026年12月31日期间发布的招标公告
- 重点关注：上半年（Q1-Q2）发布的招标需求

## 3. 核心要求（Core Requirements）

### 3.1 信息采集完整性

- [ ] 每条招标信息必须包含：**标的名称、招标单位、预算金额、关键时间节点、资质要求、核心需求描述**
- [ ] 覆盖以下采购平台：政府采购平台（中央+地方）、军队采购平台、医疗机构自主采购平台、第三方招标平台（招标信息网、千里马等）
- [ ] 最低覆盖率：目标平台100%覆盖，关键信息缺失率<5%

### 3.2 信息采集时效性

- [ ] 实时监控：关键平台每4小时轮询一次
- [ ] 增量推送：新招标公告发布后30分钟内完成抓取和入库
- [ ] 预警机制：涉及具身智能、医疗机器人、AI辅助诊断相关关键词的标的，触发实时告警

### 3.3 信息准确性

- [ ] 关键字段（预算、截止时间、资质要求）需与原文核对
- [ ] 建立信息校验机制，交叉验证多源数据一致性
- [ ] 错误率容忍度：<1%

### 3.4 数据分析深度

- [ ] 竞争分析：同品类投标竞争对手数量及名单
- [ ] 价格分析：历史同类项目成交价对比
- [ ] 趋势分析：季度/月度招标热度、品类趋势
- [ ] 关联分析：招标单位画像、评标专家库关联

|------|------|

## 5. 详细简介（Detailed Introduction）

### 5.1 情报收集体系架构

```
┌─────────────────────────────────────────────────────────────┐
│                    情报收集体系                              │
├──────────────┬──────────────┬──────────────┬───────────────┤
│  数据源层    │  采集层      │  分析层       │  应用层        │
├──────────────┼──────────────┼──────────────┼───────────────┤
│ 政府采购平台 │ 爬虫/API采集 │ 结构化处理    │ 仪表盘展示     │
│ 军队采购平台 │ 数据清洗     │ NLP实体识别   │ 商机推送       │
│ 第三方招标网 │ 数据校验     │ 关系图谱构建  │ 竞争分析报告   │
│ 行业媒体     │ 增量同步     │ 情感分析      │ 定制化查询     │
│ 企业官网     │              │ 趋势预测      │ API服务       │
└──────────────┴──────────────┴──────────────┴───────────────┘
```

### 5.2 核心功能模块

#### 5.2.1 招标信息采集模块

| 功能 | 描述 |
|------|------|
| 多源数据采集 | 支持政府集采、军队采购、第三方平台等多数据源并行采集 |
| 智能解析 | 自动解析PDF/Word/HTML格式的招标公告，提取关键字段 |
| 增量更新 | 基于时间戳和关键词的增量更新机制 |
| 全文检索 | 支持标题、摘要、全文的全文索引和检索 |

#### 5.2.2 实体识别与关联模块

| 功能 | 描述 |
|------|------|
| 实体识别 | 自动识别招标单位、中标单位、评标专家、产品型号等实体 |
| 关系抽取 | 抽取招标-中标关系、公司-人员关系、产品-技术关系 |
| 知识图谱 | 构建医疗招投标领域知识图谱，支持多跳查询 |

#### 5.2.3 商机分析与预警模块

| 功能 | 描述 |
|------|------|
| 智能匹配 | 基于公司技术能力（具身智能、机器人、AI）与招标需求智能匹配 |
| 商机评分 | 综合预算金额、竞争烈度、公司匹配度计算商机得分 |
| 实时预警 | 关键词触发、竞争对手动态、政策变化等实时告警 |
| 竞争分析 | 投标竞争对手分析、中标率统计、价格区间分析 |

#### 5.2.4 报告生成模块

| 功能 | 描述 |
|------|------|
| 日报/周报/月报 | 自动生成定期情报汇总报告 |
| 专题报告 | 行业趋势、竞争对手、目标客户画像等专题分析 |
| 自定义报告 | 支持用户自定义维度的报告生成 |

### 5.3 关键技术指标

| 指标 | 要求 |
|------|------|
| 采集平台数量 | ≥20个主流平台 |
| 信息更新频率 | 关键平台4小时轮询 |
| 召回率 | ≥95%（关键词相关招标） |
| 准确率 | ≥90%（结构化字段） |
| 响应时间 | 查询<2秒，告警<30分钟 |
| 数据容量 | 支持≥100万条招标记录 |

| 通信工程施工总承包资质 | 信息化基础设施 |
| 软件企业认定证书 | 软件产品增值税优惠政策 |

| 资质类型 | 说明 |
|---------|------|
| 电子病历系统功能应用水平分级评价 |  EMR等级（3-7级） |

| 智慧服务分级评估 | 智慧服务评级 |
| 医疗健康信息标准符合性认证 | HL7、DICOM等 |
| 等级保护2.0三级认证 | 网络安全 |

### 6.3 具身智能/机器人专项资质

| 资质类型 | 说明 |
|---------|------|
| 机器人产品认证（CR认证） | 中国机器人认证 |
| 工业机器人安全标准认证 | GB/T 15706 |
| 医疗机器人产品注册证 | 手术机器人等三类器械 |
| 康复辅助器具认证 | 康复机器人 |

| 要求 | 说明 |
|------|------|
| 政府集中采购目录内 | 需通过协议供货或集中采购 |

| 中小企业认定 | 政府采购中中小企业优惠 |
| 信用记录查询 | "信用中国"无不良记录 |

## 10. 数据交付要求

### 10.1 交付格式

| 格式 | 说明 |
|------|------|
| 结构化数据 | JSON/CSV格式，含所有字段 |
| 原始文档 | PDF/Word原版招标公告 |
| 分析报告 | Markdown/PDF格式定期报告 |
| API接口 | RESTful API，实时查询 |

### 10.2 交付频率

| 类型 | 频率 |
|------|------|
| 实时推送 | 新招标30分钟内 |
| 日报 | 每个工作日17:00 |
| 周报 | 每周一9:00 |
| 月报 | 每月第一个工作日 |
| 专题报告 | 按需 |

### 10.3 数据存储

- 存储周期：≥3年
- 备份策略：每日增量备份，每周全量备份
- 访问控制：基于角色的权限管理
